Image Gallery Component

Works in Chrome 57, Firefox 53 and Safari 10 in OSX Sierra. Feel free to test in your browsers and systems and update this page.

This is a demo page for an image gallery web component. The purpose of the component is to provide a simple means of building an image gallery without writing all the boilerplate yourself.

Something like... 

<img-gallery>
  <img src="..." alt="...">
  <img src="..." alt="...">
  <img src="..." alt="...">
</img-gallery>

Demo

The above demo shows an exmaple of custom styling for the component. By default, no styles are included; however, the component template is marked up in a way that provides fine control over all aspects of styling. See The component template below for a rendered markup reference.

Usage

Below is the source for the above component from this page.

Component API

no-deck // default: false

This tells the component not to render the deck markup. This allows you to use your own markup for the deck. It is encouraged to use the same class names from the default template for your elements so you don’t have to recreate component functionality.

back-label // default: "Previous"

This tells the component what the back button text should be.

next-label // default: "Next"

This tells the component what the next button text should be.

transition-time // default: 0

The time, in milliseconds, the component will wait before updating the deck. Set this value to the same animation time you use in your CSS.

The component template

Below is a reference of the component’s rendered template. Use this as a guide for styling. 

<img-gallery>
  <section class="img-gallery-deck">
    <section class="img-gallery-controls">
      <div class="img-gallery-controls-back">
        <button class="img-gallery-btn img-gallery-btn-back">Previous</button>
      </div>
      <div class="img-gallery-deck-items">
        <img class="img-gallery-deck-img img-gallery-deck-back" src="" alt="">
        <img class="img-gallery-deck-img img-gallery-deck-current" src="" alt="">
        <img class="img-gallery-deck-img img-gallery-deck-next" src="" alt="">
      </div>
      <div class="img-gallery-controls-next">
        <button class="img-gallery-btn img-gallery-btn-next">Next</button>
      </div>
    </section>
    <footer class="img-gallery-meta">
      <span class="img-gallery-meta-current">0</span>
      of
      <span class="img-gallery-meta-total">0</span>
    </footer>
  </section>
  <section class="img-gallery-thumbs">
    <!-- your images will be rendered here -->
  </section>
</img-gallery>

Extending the component

Since the gallery is a web component, it can be extended to create new custom components that inherit its functionality. This would allow you to implement your own versions of hooks like next(), click() and transformImageSource() to name a few.

For example, imagine you’d like to transform a gallery image source whenever its selected for display in the slide deck: 

import ImgGallery from './path/to/img-gallery';

class MyImgGallery extends ImgGallery {
  // The thumbnail images are appended with "-tn". Remove this for full-size.
  transformImageSource(img) {
    return img.getAttribute('src').replace('-tn', '');
  }
}

customElements.define('img-gallery', MyImgGallery);
